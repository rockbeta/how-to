LLM agent mechanics

How LLM-Based Agents Work

An agent is a loop around a language model: it reads a goal, reasons about the next move, calls tools, observes what happened, and keeps going until the work is done.

Explore the loop See a run

The model supplies language understanding and judgment. The agent system supplies the scaffolding: instructions, memory, tool access, validation, and a stopping rule.

That scaffolding is what turns one model response into an iterative worker that can inspect a repo, edit a file, test a change, and adapt from the result.

The core loop

A good agent is mostly disciplined repetition.

Each step narrows uncertainty. The agent moves from intent to state, from state to action, and from action back to evidence.

LLM reasoning core

Start with the goal

The user asks for an outcome. The agent translates that into a working objective, constraints, and a first guess at what evidence it needs.

Input
User request, system rules, task context
Output
A concrete objective and success criteria

What surrounds the model

Agents are systems, not just prompts.

01

Instructions

Define the role, boundaries, response style, and operating rules. They keep the model pointed at the right job.

02

Memory

Carries useful facts across steps: files read, tests run, decisions made, and details the agent should not forget.

03

Tools

Let the agent leave pure text and act on the world: search files, call APIs, run tests, edit code, browse pages, or create assets.

04

Planner

Breaks a goal into steps and updates the route when the evidence changes. Plans are useful because reality pushes back.

05

Evaluator

Checks whether the result actually satisfies the goal, often with tests, screenshots, schema validation, or a review pass.

06

Stop Rule

Decides when the work is good enough to hand back, when to ask a question, and when to stop because a constraint blocks progress.
